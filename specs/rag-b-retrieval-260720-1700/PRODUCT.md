# Phase B — RAG 检索与带来源问答

## 目标

对项目资料提问时，回答必须有来源；没有证据时明确拒答。

## 功能清单

### B-01 标题优先分块

- 按文档标题层级分组合并 Phase A 产出的小块
- 目标 500–1000 中文字符/块，相邻块 100–150 字符重叠
- XLSX 行级块保持独立（表格独立保留）
- 每块保留引用元数据（文件、标题路径、页码、工作簿+单元格范围）

### B-02 增量索引

- 基于文件 SHA-256 + 分块版本号判定是否需要重建
- 仅在源文件发生变更时才重新分块并索引
- 索引持久化到 `vector_db_root/<project_id>/`

### B-03 FAISS 向量召回

- 使用 LLM API `/v1/embeddings` 生成句子级向量
- FAISS IndexFlatIP 做余弦相似度召回，返回少量高评分候选项

### B-04 BM25 关键词召回

- `rank-bm25` 做稀疏关键词匹配
- 中文字符 unigram + bigram 辅助 tokenization
- 向量召回与关键词召回结果合并

### B-05 混合重排序 (Max 6 条)

- 对 FAISS 与 BM25 结果做 RRF (Reciprocal Rank Fusion)
- 合并 → 去重 → 归一化分数 → 取 Top 4–6
- 每条返回 `Evidence`（含 `evidence_id`、`relative_path`、`locator`、`excerpt`、`score`）

### B-06 带来源问答

- 检索到证据后构建 Prompt，要求 LLM 强制引用来
- 无检索证据时返回「未在项目资料中找到相关证据」
- 每条回答附完整来源列表

## 验收标准

完成 20 个固定问题集：

- **10 个事实题**：答案可直接在某个文件的某个段落中找到
- **5 个跨文件题**：答案需要综合至少 2 个不同文件的证据
- **5 个无答案题**：项目资料中不包含相关信息，系统应拒答

验收指标：

1. 事实题命中率 ≥ 80%（10 题中 ≥ 8 题检索到正确答案所在的 chunk）
2. 跨文件题至少召回多文件证据（不要求 LLM 回答完全正确，但检索到的 top-5 中至少含 2 个文件）
3. 无答案题全部正确返回「未在项目资料中找到」标记
4. 所有答案均附来源引用（relative_path + locator）
